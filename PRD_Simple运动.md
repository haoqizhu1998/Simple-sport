## 一、文档概述

### 1.1 文档目的

本文档为Simple运动小程序的V1.0 MVP版本产品需求文档，旨在明确产品的核心功能定义、用户交互流程、技术实现要求和版本规划，为开发团队提供完整、清晰、可执行的产品需求说明。

### 1.2 适用范围

- 开发团队：技术方案设计、功能开发、测试验收
- 设计团队：界面设计、交互设计
- 运营团队：产品上线准备、用户引导
- 相关 stakeholders：项目管理、产品评审

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| **RPE** | 主观疲劳感受量表（Rate of Perceived Exertion），1-10级评分，6=完全不累，10=极度疲劳 |
| **有氧训练** | 低强度、长时间的运动，主要消耗脂肪供能，提升耐力 |
| **间歇训练** | 高强度与恢复期交替进行的训练，提升VO2max和速度能力 |
| **恢复训练** | 低强度轻松运动，促进身体恢复，预防过度训练 |
| **VO2max** | 最大摄氧量，反映心肺功能的黄金指标 |
| **训练效果评分** | AI综合评估训练质量给出的0-100分值 |

## 二、产品概述

### 2.1 产品背景

随着大众健身意识的提升，马拉松、铁三等耐力运动参与人数持续增长。然而，普通运动爱好者面临以下痛点：

1. **数据孤岛**：运动数据分散在多个App中，缺乏统一分析
2. **专业门槛高**：专业训练理论复杂，普通用户难以理解应用
3. **决策困难**：不知道训练是否有效、不知道下一步怎么练
4. **缺乏反馈**：训练后缺少即时、专业的复盘和指导

当前市场上现有产品多为数据记录工具或社交平台，缺乏真正具备AI能力的训练决策助手。

### 2.2 产品愿景

Simple运动不是工具，而是在做**"个人运动操作系统"**：

```
数据 → 认知 → 预测 → 决策 → 再验证（闭环）
```

通过AI技术，让每个运动爱好者都能拥有"私人教练"般的训练指导体验。

### 2.3 产品定位

**AI运动决策助手** —— 通过人工智能技术，帮助用户理解训练数据、预测运动表现、做出科学决策。

### 2.4 核心切入点

**训练复盘AI（V1.0 MVP）** —— 从用户最直接、最迫切的需求出发：训练完后想知道"练得怎么样"。

### 3.1 核心用户画像

| 用户类型 | 特征描述 | 核心痛点 | 典型场景 |
|----------|----------|----------|----------|

| **大众跑者** | 跑步1-3年，以健康为目的 | 缺乏系统训练知识，不知道练得对不对 | 日常跑步后想有个"教练"点评 |

### 4.2 功能详细说明

#### 4.2.1 数据输入

**功能描述**：支持用户便捷输入运动数据

| 输入方式 | 说明 | 字段 |
|----------|------|------|
| **手动输入** | 直接填写表单 | 距离(km)、配速(min/km)、心率(bpm)、RPE(1-10) |
| **截图解析** | 上传运动App截图，AI识别提取数据 | 自动识别距离、配速、心率等关键数据 |

**手动输入字段规范**：

| 字段名 | 类型 | 必填 | 范围/格式 | 说明 |
|--------|------|------|-----------|------|
| 距离 | 数字 | 是 | 0.1-100 km | 支持小数点后1位 |
| 配速 | 时间 | 是 | 3:00-15:00 min/km | 格式 mm:ss |
| 心率 | 整数 | 否 | 80-220 bpm | 平均心率 |
| RPE | 整数 | 是 | 1-10 | 主观疲劳感受 |

**截图解析要求**：
- 支持主流运动App截图（Keep、悦跑圈、咕咚、Nike Run Club等）
- 自动框选关键数据区域
- 支持用户修正识别结果

#### 4.2.2 训练类型识别

**功能描述**：AI自动判断本次训练的类型

| 训练类型 | 判断标准 | 特征描述 |
|----------|----------|----------|
| **有氧训练** | 低强度、长距离 | 配速慢于心率区间2区，RPE 5-7 |
| **间歇训练** | 高强度、反复冲刺 | 配速快于心率区间4区，RPE 8-10 |
| **恢复训练** | 极低强度、轻松 | 配速慢，RPE 4-6，心率偏低 |

**判断逻辑说明**：
```
训练类型 = f(配速, 心率, RPE, 距离)

• 若 配速较快 + 心率较高 + RPE高 → 间歇训练
• 若 配速中等 + 心率适中 + 距离长 → 有氧训练  
• 若 配速很慢 + RPE低 + 心率低 → 恢复训练
• 若 距离短 + RPE高 + 配速快 → 冲刺/速度训练
```

#### 4.2.3 训练效果评分

**功能描述**：综合评估训练质量，给出0-100分值

| 评分区间 | 等级 | 说明 |
|----------|------|------|
| 0-40 | ⚠️ 低效/过量 | 训练质量差或运动过量，需关注 |
| 41-60 | 📊 一般 | 中规中矩，有提升空间 |
| 61-80 | ✅ 良好 | 质量不错的训练 |
| 81-100 | 🌟 优秀 | 高质量训练，状态很好 |

**评分维度**：

| 维度 | 权重 | 说明 |
|------|------|------|
| 强度匹配度 | 30% | 实际强度是否符合目标类型 |
| 时长/距离合理性 | 25% | 训练量是否合适 |
| 心率表现 | 25% | 心率是否在目标区间 |
| RPE一致性 | 20% | 主观感受与数据是否匹配 |

#### 4.2.4 是否过量判断

**功能描述**：判断本次训练是否造成运动过量

| 状态 | 判定条件 | 提示信息 |
|------|----------|----------|
| **正常** | 各项指标正常 | "训练状态良好，无过量迹象" |
| **偏累** | RPE偏高或心率偏高 | "有些疲劳，建议明天降低强度" |
| **过量** | 高RPE + 高心率 + 距离超出能力 | "⚠️ 训练过量，建议增加恢复时间" |

**过量判断规则**：
```
过量 = (RPE >= 9) AND (心率 > 年龄预估最大心率的90%) AND (距离超出近7日平均的150%)

偏累 = (RPE >= 8) OR (心率 > 最大心率的85%)
```

#### 4.2.5 能力影响分析

**功能描述**：分析本次训练对各项运动能力的提升作用

| 能力维度 | 适用训练类型 | 影响程度 |
|----------|--------------|----------|
| **耐力** | 有氧训练 | ↑ 提升 |
| **VO2max** | 间歇训练 | ↑↑ 显著提升 |
| **速度** | 冲刺训练 | ↑ 提升 |
| **恢复** | 恢复训练 | ⬆️ 促进恢复 |

**输出格式**：
```
本次训练对能力的影响：
• 耐力 ██████░░░░ +15%
• VO2max ████░░░░░░ +8%
• 速度 ███░░░░░░░ +5%
```

#### 4.2.6 明日训练建议

**功能描述**：根据本次训练情况，给出明日的训练建议

| 今日状态 | 明日建议类型 |
|----------|--------------|
| 正常/恢复训练 | "明天可以进行正常训练，建议有氧跑30-40分钟" |
| 有氧训练 | "明天建议休息或轻松慢跑，让身体恢复" |
| 间歇训练 | "今天强度较大，明天建议安排恢复跑或休息" |
| 偏累 | "建议明天完全休息或只做拉伸放松" |
| 过量 | "建议休息2-3天，注意补充营养和睡眠" |
